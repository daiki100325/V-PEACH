#  V-PEACH 要件定義書（ヌケモレ対策用）

■機能要件

毎月の各拠点の売上・経費を入力することで、

・全店舗総計

・各店個別

のPLを、

・月次

・3か月平均

・年次

で表示させたい。

  

バックエンドとなるRDBは既存アプリV-MINT2.0のSupabaseと共通化し、既存のテーブルから引っ張れる数値はそのまま引っ張ってくる。

総売上（提供売上＋物販売上）、家賃、人件費、決済手数料、光熱費、雑費、役員報酬、借入返済はV-PEACHの入力画面から打ち込む。

ただし、入力簡便化のため、総売上、物販利益率、家賃・光熱費・雑費などは、定数または定率で固定してしまい、変更が必要な場合は月次の入力画面とは別画面から変更できるようにしたい。

物販売上に関しては、各ブランドの販売数をV-MINT2.0の原価計算モードで各拠点単位で打ち込んでいるため、各ブランドの定価と掛け算すれば算出できる。総売上‐物販売上＝提供売上。

各指標について目標値を設定し、超過・余剰をハイライトして投資判断の根拠とする。

  

■アーキテクチャ

Vue 3 SPA（Vite + Tailwind CSS） + Supabase

V-MINT2.0とは別アプリとして展開するが、Supabaseのプロジェクトは共有する。

Gitにリポジトリを作成してバージョン管理およびチーム開発の基盤とする。

デプロイ先はCloudflare Pages。

  

■UI要件

デザインについては、V-MINT2.0を踏襲する。

ポータルトップから、以下のモード選択をする。

(1)PL

(2)月次入力

(3)設定

  

(1)PL

画面上部で、期間（月次/3か月平均/年次）、拠点（全店舗/馬場本店/中野店/馬場2号店）、をそれぞれ切り替えられるようにする。

一番上に、左に経費と利益、右に収益を配置した一般的なPL図。経費について、「設定」で指定した目標値とのズレをハイライトする。余剰の場合は緑、超過の場合は赤。目標値は数値または割合のどちらでも設定できるようにしたい。

  

その下に、各指標の変動を示す折れ線グラフ。月次の場合は指定した月が含まれる1年（2026年5月を参照しているときは2026年1月～2026年12月）の推移をだす。3か月平均の場合はも月次と同様（2026年1月の直近3か月平均、2026年2月の直近3か月平均、…2026年12月の直近3か月平均）。年次の場合は、各年の推移（2026年、2027年、2028年…）。

  

(2)月次入力

月次で入力が必要な項目の入力画面。V-MINT2.0の原価計算モードのUIを参考に、ステップバイステップで入力できるようにする。

  

(3)設定

以下2つのサブモードで構成される。

・機能要件で触れた、入力簡便化のための一部インプット指標の定数または定率を設定する項目

・UI要件で触れた、各経費項目の目標値を数値または割合で設定する項目
